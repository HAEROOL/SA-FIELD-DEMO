# 테스팅 전략 - SA.FIELD

## 🎭 E2E 테스트 (Playwright)
- **도구**: Playwright Native 전용.
- **모킹**: `page.route()`를 사용하여 모든 API 호출을 가로채고 고정된 데이터를 제공함.
- **효율화**: 
  - 한 페이지의 개별 데이터마다 테스트 케이스를 쪼개지 말 것.
  - "페이지 무결성" 테스트로 그룹화하여 브라우저 실행 오버헤드 최소화.
  - 복잡한 여정은 `test.step()`으로 구분.
- **반응형**: 해상도별 레이아웃 체크는 스크립트 대신 `toHaveScreenshot()` 시각적 회귀 테스트 활용.

## 🧪 통합 및 단위 테스트 (Vitest + RTL)
- **접근성 우선**: `getByRole`, `getByLabelText` 등 사용자 중심 쿼리 사용.
- **다크 모드**: UI 컴포넌트 테스트 시 `dark:` 클래스 적용 여부 확인.
- **MSW**: 통합 테스트 환경에서만 제한적으로 사용.
